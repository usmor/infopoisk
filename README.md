# TED Talks Multilingual Search Engine

Поисковая система по выступлениям TED Talks на трёх языках (русский, английский, французский) с использованием различных моделей информационного поиска.

Данные взяты из открытого источника [Kaggle](https://www.kaggle.com/datasets/miguelcorraljr/ted-ultimate-dataset)

## О проекте

Проект реализует мультиязычный поиск по корпусу TED Talks. Особенности:
- **3 языка**: русский, английский, французский
- **3 модели поиска**: BM25, Word2Vec, FastText
- **Автоопределение языка** запроса
- **Группировка результатов** по выступлениям (показываются все доступные переводы)
- **Единый интерфейс** для всех моделей

## Этапы работы 
1. Скачивание данных и сведение их к словарной структуре (объединение транскриптов по `talk_id`) –– `data_loader.py`
2. Обработка полученного корпуса трансриптов (`spacy` –– для английского и французского; `pymorphy3` –– для русского) –– `preprocessing.py`
3. Создание базового класса индекса –– `base_index.py`
4. Последовательное создание классов индексов BM25, Word2Vec, FastText –– `bm25_index.py`, `word2vec_index.py`, `fasttext_index.py`
5. Создание класса для быстрого получения документов (транскриптов и метаданных) –– `data_storage.py`
6. Общий для всех индексов класс поиска (CLI) –– `cli_search.py`
7. Точка входа CLI (парсинг запроса, форматирование результатов, вывод) –– `cli.py`

## Автоопределение языка
Система автоматически определяет язык запроса с помощью библиотеки `langdetect`: русский, английский, французский.

Язык запроса влияет на:

- Выбор стоп-слов при лемматизации

- Приоритет языков в результатах

- Язык сообщений интерфейса

## Модели поиска
### 1. BM25
Тип: Вероятностная модель ранжирования

Источник: Библиотека `rank-bm25`

Описание: Модель на основе частоты терминов и обратной частоты документа

Параметры: `k1=1.5, b=0.75`

### 2. Word2Vec
Тип: Статическая нейросетевая модель

Источник: Обучена на корпусе TED Talks с помощью `gensim`

Описание: Векторные представления слов для семантического поиска

Параметры: `vector_size=300, window=5, min_count=2, epochs=10`

### 3. FastText
Тип: Статическая модель с учётом подслов (subword information)

Источник: Обучена на корпусе TED Talks с помощью `gensim`

Описание: Учитывает морфологию через n-граммы символов

Параметры: `vector_size=300, window=5, min_count=2, epochs=10, min_n=3, max_n=6`

## Установка

### 1. Клонирование репозитория
```bash
git clone <URL_репозитория>
```

### 2. Создание виртуального окружения
```bash
python -m venv venv

# Активация (Linux/Mac)
source venv/bin/activate

# Активация (Windows)
venv\Scripts\activate
```

### 3. Установка зависимостей
```
pip install -r requirements.txt
```

### 4. Скачивание данных
В папке `data` лежат исходные датасеты, обработанные загрузить в гит не удалось в виду их веса.
Их нужно скачать и положить в папку `data`.
- [ted_documents.json](https://drive.google.com/file/d/1Vw-PYnBweNh6BkRDNnMtELo4723uqBZ2/view?usp=sharing)
- [ted_preprocessed.json](https://drive.google.com/file/d/19dZL__iHJWIDORLVQocsdVQiPG_O6whA/view?usp=sharing)

### 5. Построение индексов
```
cd backend/src
python build_indexes/build_all_indexes.py 
```

## Поиск через CLI
### 1. Переходим в нужную директорию
``` bash
cd backend/src
```

### 2. Запускаем файл CLI с запросом
``` bash
python cli.py -q <ЗАПРОС> -i <ИНДЕКС> -k <МАКС. КОЛ-ВО РЕЗУЛЬТАТОВ>
```

### Формат запросов и параметры
Обязательные параметры:
- `-q, --query`	––– текст поискового запроса (на любом из трёх языков)

Опциональные параметры:
- `-i, --index`	––– тип поискового индекса (по умолчанию bm25)
- `-k, --top-k` –––	количество результатов (по умолчанию 10)

### Пример работы
Запрос: 
```bash
python cli.py -q "В чем смысл жизни?" -i word2vec -k 5
```
