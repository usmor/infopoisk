# Homework 1
Система поиска по корпусу русских сказок с использованием различных типов обратных индексов: частотных и BM25, с реализацией через библиотеки и собственные структуры данных.

## Данные
Исходный датасет был взят на Kaggle ([ссылка](https://www.kaggle.com/datasets/idoldev/adult-and-child-russian-tales-dataset-with-label?resource=download)). Я брала только детские сказки (лейбл = 1), так как сказки для иных возрастов не похожи на сказки и содержали неприличный контент. Я шла при выборе датасета по пути меньшего сопротивления и выбрала один из первых открытых датасетов. А сказки -- это просто приятный бонус: в них меньше всего ожидаешь гиперссылки, странные символы и прочие "препятствия" нормального препроцессинга.

Препроцессинг включал:
- приведение текста к нижнему регистру и токенизацию;
- удаление неалфавитных символов;
- лемматизацию через `pymorphy3`;
- удаление стоп-слов и очень коротких слов (≤2 символов).

В итоговом корпусе 1030 текстов.

## Установка

1. Клонируйте репозиторий:
```bash
git clone <URL_репозитория>
cd homework1
```

2. Создайте и активируйте виртуальное окружение:

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

## Использование поиска

В проекте есть универсальная функция поиска `search`, которую можно запустить `через main.py`. Программа попросит:
- Ввести поисковый запрос
- Выбрать тип индекса: `freq` или `bm25`
- Выбрать подтип индекса: `lib`, `dict` или `matrix`
- Количество результатов для отображения

В результате будет выведен текст (ограничен до 1000 знаков для удобства вывода) и его score.

## Построение индексов
При запуске `build_indexes/main.py` будет совершен повторный препроцессинг корпуса и будут созданы все индексы заново.

_Примечание_: поиск в алгоритмах, реализованный руками, менее строгий, чем поиск через библиотеку.
